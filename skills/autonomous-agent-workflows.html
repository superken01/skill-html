自主代理工作流

用 Hermes、Codex、Claude Code、OpenCode 等代理拆分任務、平行執行與驗收。

多代理HermesCodexClaude Code工作流

中文整理

這個技能是一套多代理開發方法：先由主代理界定目標與完成定義，再把可獨立處理的子任務委派給其他代理或 CLI，最後由主代理整合、驗證、提交。它強調隔離、可觀測、可回復，而不是讓多個代理同時亂改同一個工作區。

何時使用

  • 任務可以拆成多個互不衝突的子工作，例如前端、後端、測試、文件或調研。
  • 需要快速探索多個方案，或讓一個代理實作、另一個代理 review。
  • 需要長時間背景工作，但仍要主代理掌握狀態與最終品質。

不建議使用情境

  • 任務很小，單一代理直接完成比較快。
  • 變更高度耦合，平行實作會造成 merge 衝突。
  • 沒有測試或驗收標準，代理結果無法可靠判斷。

建議流程

  • 定義成果：列出要交付的檔案、行為、測試與不做事項。
  • 拆分任務：每個子任務都應有清楚輸入、輸出與邊界。
  • 選擇執行方式：短任務可用 Hermes delegation；需要完整 CLI 能力時用 Codex/Claude Code/OpenCode。
  • 隔離工作區：使用 branch、worktree 或獨立 repo clone，避免代理互踩。
  • 收斂結果：讀取每個代理摘要，但以實際 diff、測試與檔案內容為準。
  • 整合與驗證：主代理負責最後修整、跑測試、commit、PR 與回報。

命令範例

Hermes 一次性子代理

hermes chat -q "請檢查這個 repo 的登入流程並寫出風險摘要。"

使用 worktree 隔離平行開發

git worktree add ../feature-a -b feat/feature-a

整合前檢查

git diff --stat && git status --short

注意事項

  • 多代理不是免審核。主代理仍要檢查每個結果是否符合需求。
  • 任務描述要包含完成定義，否則代理會用自己的假設填空。
  • 能用小步驗證就不要一次丟巨大任務；可觀測性比一次生成大量程式碼更重要。
