01

用途與工作方式

對應來源章節：Obsidian Vault

這個 skill 用於以檔案系統優先的方式操作 Obsidian vault：讀取筆記、列出筆記、搜尋 Markdown 檔、建立筆記、追加內容，以及加入 Obsidian wikilinks。

核心原則

先解析出具體 vault 絕對路徑，再使用 Hermes 的 file tools。不要把含有 $OBSIDIAN_VAULT_PATH 的字串直接傳給 read_file、write_file、patch 或 search_files。

02

Vault path 路徑慣例

對應來源章節：Vault path

在呼叫檔案工具前，必須先知道或解析 vault path。文件化的慣例是使用環境變數 OBSIDIAN_VAULT_PATH，例如放在 ${HERMES_HOME:-~/.hermes}/.env。若沒有設定，fallback 是 ~/Documents/Obsidian Vault。

  • 檔案工具不會展開 shell 變數，所以不要把 $OBSIDIAN_VAULT_PATH 直接當路徑傳入。
  • Vault path 可能包含空白，因此解析出絕對路徑後，優先使用 read_file / write_file / patch / search_files，而不是 shell 指令。
  • 如果 vault path 不明，可以先用 terminal 解析環境變數或檢查 fallback path 是否存在；一旦路徑確定，就切回檔案工具。

本機建議設定

OBSIDIAN_VAULT_PATH="/Users/super/Library/Mobile Documents/iCloud~md~obsidian/Documents/Hermes"
03

讀取與列出筆記

對應來源章節：Read a note / List notes

讀取單一筆記時，使用 read_file 搭配已解析的絕對路徑。這比 cat 更適合，因為會提供行號與分頁。

讀取筆記

read_file(path="/absolute/path/to/vault/Note.md")

列出所有 Markdown 筆記

search_files(pattern="*.md", target="files", path="/absolute/path/to/vault")

列出子資料夾筆記

search_files(pattern="*.md", target="files", path="/absolute/path/to/vault/Subfolder")
04

搜尋筆記

對應來源章節：Search

檔名搜尋與內容搜尋都使用 search_files。來源 skill 明確建議用它取代 grep、find 或 ls。

依檔名搜尋

search_files(pattern="*project*.md", target="files", path="/absolute/path/to/vault")

搜尋 Markdown 內容

search_files(pattern="關鍵字或 regex", target="content", path="/absolute/path/to/vault", file_glob="*.md")
05

建立筆記

對應來源章節：Create a note

建立筆記時，使用 write_file 傳入解析後的絕對路徑與完整 Markdown 內容。這比 shell heredoc 或 echo 更安全，能避免 shell quoting 問題，也會回傳結構化結果。

建立 Markdown 筆記範例

write_file(
  path="/absolute/path/to/vault/Project.md",
  content="# Project\n\n- 相關：[[Another Note]]\n"
)
06

追加內容與精準編輯

對應來源章節：Append to a note / Targeted edits

  1. 優先讀取目標筆記：read_file。
  2. 若有穩定 anchor，例如既有標題或固定尾端區塊，用 patch 把 anchor 替換成 anchor + 新內容。
  3. 如果整份重寫比脆弱 patch 更清楚，可以用 write_file 重寫完整筆記。
  4. 若只是沒有穩定上下文的簡單 append，且 file-tool workflow 太彆扭，才考慮用 terminal。

針對局部內容修改時，用 patch 做 focused edit，避免用 shell 文字改寫。

07

Obsidian Wikilinks

對應來源章節：Wikilinks

Obsidian 使用 [[Note Name]] 語法連結筆記。建立新筆記時，應用 wikilinks 連到相關主題，讓 vault 可以形成可瀏覽的知識網。

Wikilink 範例

# Hermes Notes

相關：[[Project Decisions]]、[[Daily Log]]

支援檔資訊

這些是來源 skill 目錄中 `SKILL.md` 以外的檔案；HTML 只列出檔名與用途提示，不直接貼整份英文內容。

來源 skill 沒有額外支援檔，或本機未找到。
