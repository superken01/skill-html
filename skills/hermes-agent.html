中文整理

Hermes Agent 是 Nous Research 的開源 AI 代理框架。它可以在終端機、通訊平台與 IDE 中使用，透過工具呼叫操作檔案、執行命令、搜尋資料、管理記憶與技能。這份中文整理著重在日常使用與設定：profile、skills、tools、MCP、gateway、cron、memory，以及遇到問題時的排查方向。

何時使用

  • 需要設定 Hermes CLI、模型、provider、tools、skills 或 profile。
  • 要把 Hermes 接到 Telegram、Discord、Slack、Webhook、MCP server 或排程工作。
  • 要了解 Hermes 如何保存記憶、載入技能、委派子代理，或如何擴充工具。

不建議使用情境

  • 只是一般 repo 程式碼問題，且不涉及 Hermes 本身設定或平台能力。
  • 需要最新權威資訊但本機 skill 可能過期；這時應優先查官方文件。

建議流程

  • 先確認 profile：不同 profile 有獨立 config、skills、sessions 與 memory。
  • 檢查狀態：用 hermes doctor、hermes status、hermes config path 找設定與依賴問題。
  • 設定模型與認證：用 hermes model 或 hermes auth add 管理 provider 與 credential。
  • 管理工具與技能：用 hermes tools list、hermes skills list、hermes skills install 檢查可用能力。
  • 整合外部系統：MCP 用 hermes mcp add/list/test；gateway 用 hermes gateway setup/run/install。
  • 變更後重啟或 reset：工具、技能與部分設定通常要新 session 或 gateway restart 才生效。

命令範例

健康檢查

hermes doctor

查看目前設定檔路徑

hermes config path

切換或設定模型

hermes model

列出工具與技能

hermes tools list && hermes skills list

測試 MCP server

hermes mcp list && hermes mcp test <name>

注意事項

  • 官方文件 https://hermes-agent.nousresearch.com/docs 是最新權威來源；本機 skill 是常用流程摘要。
  • profile 很重要。dev、default、pm 等 profile 的資料夾不同，不要誤改別的 profile。
  • 工具變更通常不會套用到已存在的對話，開新 session 或 reset 才能看到差異。
