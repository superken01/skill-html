01

定位與差異化

對應來源章節：Hermes Agent

Hermes Agent 是 Nous Research 的開源 AI agent framework，可在 terminal、messaging platforms、IDEs 中執行。它和 Claude Code、Codex、OpenClaw 同屬 autonomous coding/task-execution agents，透過 tool calling 操作系統。Hermes 支援 OpenRouter、Anthropic、OpenAI、DeepSeek、本地模型等多種 provider，並可在 Linux、macOS、WSL 執行。

  • 透過 skills 自我改善：把成功流程、被修正的做法與可重用程序存成 skill。
  • 跨 session persistent memory：保存偏好、環境與 lessons learned，可用不同 memory backends。
  • Multi-platform gateway：同一 agent 可跑在 Telegram、Discord、Slack、WhatsApp、Signal、Matrix、Email 等平台。
  • Provider-agnostic：可切換 model/provider，credential pools 可輪替多把 API keys。
  • Profiles：多個 Hermes instance 有獨立 configs、sessions、skills、memory。
  • Extensible：plugins、MCP servers、custom tools、webhooks、cron、Python ecosystem。
權威文件

最新資訊以官方 docs 為準：https://hermes-agent.nousresearch.com/docs/

安全與隱私切換

對應來源章節：Security & Privacy Toggles

  • Secret redaction 預設開啟：hermes config set security.redact_secrets true。若要關閉通常需新 session，且只應在 redactor/debug 情境使用。
  • Gateway PII redaction：hermes config set privacy.redact_pii true|false。
  • Command approvals：approvals.mode 可為 manual、smart、off；--yolo 是 per-invocation bypass。
  • Shell hooks allowlist 由 ~/.hermes/shell-hooks-allowlist.json 管理。
  • 若要避免 web/browser/image-gen，透過 hermes tools 對 platform 關閉，下一 session 生效。
08

Voice、Spawning 與 Durable Systems

對應來源章節：Voice & Transcription / Spawning Additional Hermes Instances / Durable & Background Systems

STT provider 偵測順序包含 local faster-whisper、Groq Whisper、OpenAI Whisper、Mistral Voxtral；TTS provider 包含 Edge TTS、ElevenLabs、OpenAI、MiniMax、Mistral、NeuTTS。互動中用 /voice on|tts|off。

額外 Hermes instances 可用 one-shot hermes chat -q 或 tmux interactive PTY。短期子任務偏向 delegate_task；長期或完全獨立 process 可 spawn Hermes，搭配 -w worktree mode 降低 git conflict。

  • Delegation：同步 child agent spawn，不 durable。
  • Cron：durable scheduler，支援 duration、every phrase、cron、ISO timestamp。
  • Curator：背景 skill lifecycle 維護，只 archive、不硬刪。
  • Kanban：SQLite board，dispatcher claims tasks 並 spawn assigned profiles。

Troubleshooting 與去哪裡找資料

對應來源章節：Troubleshooting / Where to Find Things

  • Voice 不動：檢查 stt.enabled、provider dependencies/API key、gateway restart。
  • Tool 不存在：檢查 hermes tools、環境變數，啟用後 /reset。
  • Model/provider 問題：跑 hermes doctor、重新 hermes auth、檢查 .env。
  • 變更未生效：tools/skills 需要 /reset，gateway 需要 /restart，code change 需重啟 CLI/gateway。
  • Skills 不顯示：hermes skills list、hermes skills config、或顯式 /skill name。
  • Gateway 問題：先查 ~/.hermes/logs/gateway.log。

Contributor quick reference

對應來源章節：Contributor Quick Reference

Source skill 提供偶爾 contributor/PR author 的開發速查：主要檔案包含 run_agent.py、model_tools.py、toolsets.py、cli.py、hermes_state.py、agent/、hermes_cli/、tools/、gateway/、cron/、tests/、website/。

  • 新增 tool 通常涉及 tools/your_tool.py 與 toolsets.py，並使用 registry.register。
  • 新增 slash command：在 hermes_cli/commands.py 加 CommandDef，並補 CLI/gateway handler。
  • 測試：python -m pytest tests/ -o 'addopts=' -q 或針對特定 area。
  • Commit conventions：fix:、feat:、refactor:、docs:、chore:。
  • 關鍵規則：不要破壞 prompt caching、維持 message role alternation、路徑使用 get_hermes_home()、secrets 放 .env、tools 要有 check_fn。
