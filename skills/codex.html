中文整理

這個技能用來啟動與管理 Codex CLI，讓它在獨立工作目錄中完成明確的開發任務。重點不是把需求丟出去就放任不管，而是先界定範圍、提供必要上下文、要求測試與提交紀錄，再由 Ethan 檢查輸出與整合結果。

何時使用

  • 有明確可切分的程式任務，例如新增一個 endpoint、修一個錯誤、補測試或整理文件。
  • 需要讓另一個代理平行探索或實作，但仍希望最後由目前工作流審核。
  • 想用 Codex 擅長的 repo 內編輯與命令執行能力，加速重複性開發工作。

不建議使用情境

  • 需求還很模糊，尚未定義成功條件或檔案範圍。
  • 任務會改動憑證、部署、付款或刪除資料等高風險資源。
  • 目前 repo 有未提交的重要變更，尚未隔離 branch 或 worktree。

建議流程

  • 確認 repo 狀態：先看 git status，避免 Codex 覆蓋人工未提交變更。
  • 建立清楚任務說明：包含目標、可改檔案、不可改事項、測試命令、完成回報格式。
  • 以 Codex CLI 執行：短任務可前景執行，長任務用背景程序並開啟完成通知。
  • 讀取 Codex 回報與 git diff：不要直接相信摘要，要實際檢查變更。
  • 跑測試與 lint：至少執行任務相關測試，必要時跑完整驗證。
  • 由 Ethan commit 或要求 Codex 產出可審核 commit，再進 PR workflow。

命令範例

一次性委派範例

codex exec --full-auto "請在目前 repo 實作登入錯誤訊息修正，先跑相關測試，最後回報變更與測試結果。"

指定模型與工作目錄

cd /path/to/repo && codex exec --model gpt-5.1-codex --full-auto "依照 issue 描述補上單元測試與修正。"

檢查 Codex 造成的變更

git status --short && git diff --stat && git diff

注意事項

  • Codex 的輸出只是候選實作；完成前仍要由目前代理做 code review 與本地驗證。
  • 任務 prompt 要明確要求不要改無關檔案，並要求列出測試命令與結果。
  • 若需要與 Hermes 多代理並行，優先使用隔離 worktree 或獨立 branch，避免互相覆蓋。
